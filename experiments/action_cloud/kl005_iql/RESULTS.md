### 1.3 核心发现

1. **z_center 在 topdown 环境大幅下降**（3.4→2.3-2.5），说明云变宽后 Actor 确实更靠近云心了。但在 mix 环境下降很少（2.7→2.6），改善有限。

2. **adv 在 mix 环境变得持久**（100k 仍未死亡，q90≈0.09），这与 baseline（5k-11k 死亡）显著不同。但 adv 没死不代表 reward 没崩——mix_b0/b5/b8 的 reward 仍然崩塌。

3. **peak reward 提升**（mix 从 160→200），说明上升期更猛了。

4. **但 reward 崩塌仍在大多数 run 中发生**：8 个有完整数据的 run 中，5 个崩塌（drop >40%），只有 mix_b2 和 td_b0 基本不崩。

5. **关键矛盾**：z_center 降了、adv 活得久了、peak 更高了，但 reward 还是崩。说明云宽度不是唯一的瓶颈——还有其他因素（BC 兜底、Actor 归一化等）也需要修复。

## 四、结论

### 云宽度确实改善了

1. **z_center 在 topdown 下降 ~1.0**（3.4→2.4）——这是幅度最大的改善。云变宽 → Actor 能靠近云心。
2. **adv 在 mix 变得非常持久**——100k 步仍未死亡，彻底改变了"adv 快速耗尽"的 baseline 动力学。
3. **peak reward 更高**——说明上升期收益更大了。

### 但宽度不是全部

4. **reward 在大多数 run 中仍然崩塌**。z_center 降了、adv 活得久了、peak 高了，但终值还是会崩。这说明：
   - **云宽度是必要条件，但不是充分条件**——Actor 离云近了，但它需要有正确的方向才能往里走，而 AWR 在 adv 弱化后给不了这个方向
   - **BC 兜底（λ_bc=0）仍然缺失**——即使 adv 活得久，一旦 adv 消失，没有 BC 把 Actor 固定在云内

### 两个意外

5. **mf_init 的 td_b0 表现极好**（终值 154，跌 7%）。需要理解为什么 MF embedding 在 topdown 上效果这么好——可能是 MF 的 item 表示（E_cos 更低）在 tdPBM 环境下起作用。
6. **z_center 在 mix 下降很少**（2.7→2.6），跟 topdown 的大幅下降形成鲜明对比。mix 和 topdown 的云结构不同导致同样的 KL=0.05 改善效果不对称。

### 下一步

- 补跑 mf_init 中 crash 的 5 个 run
- 验证 td_b0 mf_init 的优异结果是否可复现
- 考虑加 BC 兜底（λ_bc=0.3）+ KL=0.05 的组合实验
